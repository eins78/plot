## Key decisions

- **Single-plugin marketplace stays single-plugin.** Plot's sub-skills (`plot-idea`, `plot-approve` etc.) are subcommands of one workflow, not separately installable plugins. Generating per-skill marketplace entries (as agent-skills does) would mis-shape distribution. Skill-level `metadata.version` bumping is still wired up because `CLAUDE.md` mandates it.
- **Forward-compatible marketplace filter** (`.plugins |= map(.version = $v)` instead of `.plugins[0].version = $v`) ŌĆö costs nothing today, no rework if plot ever splits into multiple plugin entries. Suggested by advisor.
- **Kept the local `pnpm run release` escape hatch** even though CI handles releases ŌĆö useful when Actions is broken. Suggested by advisor.
- **Kept self-attribution stripping** in `sync-versions.sh` ŌĆö harmless, already validated upstream.

## Open notes for the user

The first changeset-driven release will graduate the pre-release suffix and produce a section header ŌĆö likely `## 1.0.0` again (if patch) or `## 1.1.0` (if minor). 3. **`eins78/plot-marketplace` may need a downstream update** once a release lands here. The release pipeline in this repo does **not** touch `plot-marketplace`. Document that integration if/when it becomes load-bearing.
4. **`pnpm` update notice** (`10.27.0 ŌåÆ 10.33.2`) appeared during install ŌĆö pinned in `packageManager`, not a blocker.
5. **First-time release UX:** initial release PR will be `release: 1.0.0` (or whatever the first bump produces). The prior CHANGELOG content can be merged into that PR's autogenerated entry, or kept as a separate "previously released" section.
